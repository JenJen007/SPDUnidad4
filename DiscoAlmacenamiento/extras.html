Acceso e Interfaces

Las unidades de disco duro se conectan a las computadoras a través de diversos tipos de interfaces, incluyendo ATA paralelo, ATA serie, SCSI, SAS (SCSI conectado en serie), canal de fibra, IEEE 1394, USB, entre otros. Estas interfaces son digitales y permiten la comunicación entre la unidad y el sistema, independientemente de cómo se codifican los datos internamente en la unidad o cuántos discos y cabezales físicos tenga la unidad.

Para procesar las señales analógicas de los cabezales de lectura/escritura, las unidades modernas utilizan un DSP (Procesador de Señal Digital) que emplea técnicas como PRML y corrección de errores Reed-Solomon. Esto permite decodificar los datos y enviarlos a la interfaz estándar. El DSP también monitorea la tasa de errores, realiza reasignaciones de sectores defectuosos y recopila datos para funciones de autodiagnóstico y análisis.

Las interfaces modernas utilizan un solo cable de datos/control para conectar la unidad al host, junto con un cable de alimentación directo a la fuente de alimentación. En contraste, las interfaces más antiguas requerían cables separados para señales de datos y control.

Algunos de los estándares de interfaz más conocidos incluyen:

  • SCSI (Small Computer System Interface): Utilizado en servidores, estaciones de trabajo y computadoras como Amiga, Atari ST y Macintosh. Aunque muchas computadoras han migrado a interfaces más nuevas, el conjunto de comandos SCSI todavía se utiliza en la interfaz SAS moderna.
  • IDE (Integrated Drive Electronics), también conocido como ATA (Advanced Technology Attachment): Movió el controlador HDD de la tarjeta de interfaz a la unidad de disco, lo que estandarizó la interfaz host/controlador y redujo la complejidad. IDE utilizaba cables de 40 pines, que posteriormente se actualizaron a cables de 80 conductores para admitir modos de alta velocidad como UDMA.
  • EIDE (Enhanced Integrated Drive Electronics): Una mejora no oficial de IDE que introdujo DMA para transferir datos directamente entre la memoria y el disco sin la intervención de la CPU.
  • Fibre Channel (FC): Un sucesor de la interfaz SCSI en el ámbito empresarial, utilizado en redes de almacenamiento (SAN). Aunque se llama "Fibre," a menudo se utiliza cobre en lugar de fibra óptica para conexiones de unidad.
  • SAS (Serial Attached SCSI): Un protocolo de comunicación serie que permite transferencias de datos a alta velocidad y es compatible con SATA. Utiliza una conexión mecánicamente compatible con discos duros SATA estándar.
  • SATA (Serial ATA): Utilizado comúnmente en computadoras personales, utilizan conexiones serie para la transmisión de datos. Las unidades SATA pueden ser controladas por controladores SAS en algunos casos, pero no al revés.

Interfaces de Disco Duro

  • IDE (Integrated Drive Electronics): IDE integra el controlador de disco duro con el propio disco. Esto reduce la cantidad de cables y mejora la confiabilidad de la transmisión de datos. Aunque es económico y compatible, se ha vuelto menos común en favor de interfaces más modernas.
  • SCSI (Small Computer System Interface): SCSI es una interfaz de alta velocidad que se utiliza principalmente en servidores y estaciones de trabajo avanzadas. Ofrece ventajas como multitarea, alto ancho de banda y capacidad de intercambio en caliente, pero es más costoso que IDE.
  • Canal de fibra (Fiber Channel): Inicialmente diseñado para redes, el canal de fibra se utiliza en sistemas de disco duro de alto rendimiento. Ofrece características como intercambio en caliente, ancho de banda de alta velocidad y conexión de larga distancia.
  • SATA (Serial ATA): SATA es una interfaz de disco duro serial utilizada en la mayoría de las computadoras personales. Ofrece ventajas como mayor capacidad de corrección de errores y velocidades de transmisión más rápidas que IDE.
  • SATA II: SATA II es una versión mejorada de SATA que aumenta la velocidad de transmisión externa de 1.5Gbps a 3Gbps y presenta tecnologías adicionales como NCQ (Native Command Queuing).

Controlador Raid

Un controlador RAID es una tarjeta o chip ubicado entre el sistema operativo y una unidad de almacenamiento (generalmente un disco duro).

¿Qué hace un controlador RAID?

Virtualiza las unidades en diferentes grupos con características específicas de protección de datos y redundancia. La interfaz front-end normalmente se comunica con el servidor a través de un adaptador basado en host (HBA). El backend se comunica con el medio de almacenamiento subyacente y lo administra; generalmente es ATA, SCSI, SATA, SAS o Fibre Channel.

Los controladores RAID se clasifican por las múltiples características que incluyen tipos de unidades (como SATA o SAS), niveles RAID específicos y el número de puertos y unidades compatibles. El controlador RAID no es un controlador de almacenamiento. El controlador de almacenamiento proporciona el disco activo al sistema, mientras que el controlador RAID actúa como caché de RAM y proporciona funciones RAID.

Controladores RAID de hardware VS software

  • Basado en hardware: Los controladores RAID de hardware son dispositivos dedicados diseñados para gestionar las configuraciones RAID de un conjunto de discos. Existen dos variantes principales: tarjetas controladoras RAID y RAID en chip interno.
  • Tarjetas controladoras RAID: Son dispositivos físicos que se insertan en las ranuras PCIe o PCI-X de una placa base. Estas tarjetas incorporan su propio procesador RAID y procesador de E/S con interfaz de unidad independiente, lo que les permite administrar y controlar completamente los discos conectados.
  • RAID en chip: Esta opción más económica implica la integración de un chip RAID en la placa base. Este chip único incluye una interfaz de host, una interfaz de E/S para los discos duros, un procesador RAID y un controlador de memoria.
  • Basado en software: Los controladores RAID de software operan desde el sistema host en lugar de un dispositivo físico independiente
  • RAID solo de software: Este enfoque utiliza las funciones RAID nativas del sistema operativo anfitrión y se considera la opción más económica. El cálculo de RAID y la gestión de discos se realizan a través de la aplicación basada en el host,utilizando controladores nativos de host bus adapter (HBA) o interfaces de E/S para comunicarse con las unidades de almacenamiento.
  • RAID híbrido de hardware/software: Este enfoque combina las funciones de BIOS RAID proporcionadas por la placa base o un HBA con un componente de hardware adicional. Esta tecnología híbrida agrega una capa adicional de administración y, aunque aumenta el costo en comparación con el RAID solo de software, ofrece protección contra errores de inicio del sistema operativo.
